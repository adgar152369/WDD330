Validating Form and Using Fetch

What is Form Validation?

  • Browser and/or web server will check to see if the data entered is in the correct format and within the constraints of the application
  • Client-side validation is done in the browser, and server-side validation is done on the server
  • If info is entered correctly, app allows the data to be submitted to the server

Goal of Validation

  • Get the correct data, in the correct format
  • Protect usersâ€™ data
  • Protect ourselves

Types of Client-side validation

  • Built-in HTML5: better performance and not as customizable
  • JavaScript is customizable

The Constraint Validation API

  • Set of methods and properties available on the following form DOM interfaces:
  • button
  • textarea
  • input, etc.
  • The API makes the following properties available:
  • validationMessage
  • Validity
  • ValidityState
  • willValidate
