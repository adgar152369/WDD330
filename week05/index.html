Exceptions

  • An error that produces a return value which can be used by program to deal with the error
  • Exception will produce a stack trace
  • Strack trace is a sequence of function or method calls that leads to where the error occured
  • Will work backwards from the point at which the error occurred to identify the function that started the sequence

Warnings

  • Error in the code isn't enough to make a program crash
  • Program will continue to run
  • Program can run incorrectly

Strict Mode

  • Increases chance of errors
  • This helps to spot errors early on
  • Required to use strict mode: 'use strict';
  • Instead of a warning, you will get an exception

Linting Tools

  • Can be used to test the quality of JavaScript code, beyond simply using strict mode
  • Passing a lint test is no guarantee that your code is correct, but it will mean it will be more consistent and less likely to have problems

Debugging in the Browser

  • The point at which an error occurs is not always where it originated, so you’ll need to run through the program to see what’s happening at different stages of its execution
  • Can be useful to create what are known as breakpoints
  • JavaScript environments have a console object that provides a number of methods for logging information and debugging

Exception Handling

  • When an exception occurs, the program terminates with an error message
  • If we suspect a piece of code will result in an exception, we can wrap it in a try block
  • Any code placed inside a try block will pass any error objects to a catch block when an exception occurs.
